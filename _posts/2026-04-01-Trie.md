## The Problem

**[LeetCode 208 - Implement Trie (Prefix Tree)](https://leetcode.com/problems/implement-trie-prefix-tree)**

Implement a Trie with the following operations:

- `insert(word)` — Insert a word into the trie.
- `search(word)` — Return `true` if the word exists in the trie.
- `startsWith(prefix)` — Return `true` if any word in the trie starts with the given prefix.

```
Trie trie = new Trie();
trie.insert("apple");
trie.search("apple");     // true
trie.search("app");       // false
trie.startsWith("app");   // true
trie.insert("app");
trie.search("app");       // true
```

---

## What is a Trie?

A **Trie** (pronounced "try", short for *retrieval*) is a tree-shaped data structure designed for fast string lookups.

Unlike a hash map that stores whole strings as keys, a Trie breaks each word into individual characters and stores them along a path from the root to a leaf. Characters that share a common prefix also share a common path.

```
Words: ["apple", "app", "apt", "bat"]

         root
        /    \
       a      b
       |      |
       p      a
      / \     |
     p   t    t
     |   |
     l   (end)
     |
     e
    (end)
```

- "app" and "apple" share the path `a → p → p`
- "apt" branches off after `a → p`
- "bat" is completely separate under `b`

Each node stores up to 26 children (one per lowercase letter) and a boolean flag marking whether a complete word ends there.

---

## Why use a Trie?

| Operation | Hash Map | Trie |
|-----------|----------|------|
| Insert | O(m) | O(m) |
| Search (exact) | O(m) | O(m) |
| Prefix search | O(n·m) — scan all keys | O(m) — follow the path |

*m = word length, n = number of words*

The key advantage is **prefix search**. A hash map has no structure around shared prefixes — to find all words starting with "app" you'd have to scan every key. A Trie lets you jump directly to the "app" node and everything beneath it is your answer.

---

## Node Structure

Each Trie node needs:
1. An array of 26 child pointers (one per letter `a–z`)
2. A boolean `isEnd` to mark the end of a complete word

```java
class TrieNode {
    TrieNode[] children = new TrieNode[26];
    boolean isEnd = false;
}
```

The index for a character is computed as `c - 'a'`:
- `'a'` → index 0
- `'b'` → index 1
- `'z'` → index 25

A `null` child means no word passes through that branch.

---

## Approaching the Problem Step by Step

### Insert: "apple"

Start at the root. For each character, check if the child exists — create it if not — then move down.

```
insert("apple")

root
 → 'a' (index 0): no child → create node
    → 'p' (index 15): no child → create node
       → 'p' (index 15): no child → create node
          → 'l' (index 11): no child → create node
             → 'e' (index 4): no child → create node
                → mark isEnd = true
```

### Insert: "app"

```
insert("app")

root
 → 'a': exists → move down
    → 'p': exists → move down
       → 'p': exists → move down
          → mark isEnd = true   ← "app" ends here
```

The path to "app" was already built by "apple" — we just flip the `isEnd` flag.

### Search: "app"

Walk the path character by character. If we fall off (a child is `null`), return `false`. If we reach the end, check `isEnd`.

```
search("app")
 → 'a': exists
 → 'p': exists
 → 'p': exists, isEnd = true → return true
```

### startsWith: "ap"

Same walk as `search`, but we don't check `isEnd` — we only care that the path exists.

```
startsWith("ap")
 → 'a': exists
 → 'p': exists → return true   ← prefix found
```

## The Code

```java
class Trie {

    private TrieNode root;

    public Trie() {
        root = new TrieNode();
    }

    public void insert(String word) {
        TrieNode node = root;
        for (char c : word.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) {
                node.children[idx] = new TrieNode();
            }
            node = node.children[idx];
        }
        node.isEnd = true;
    }

    public boolean search(String word) {
        TrieNode node = traverse(word);
        return node != null && node.isEnd;
    }

    public boolean startsWith(String prefix) {
        return traverse(prefix) != null;
    }

    // Shared helper: walk the trie along the given string.
    // Returns the node at the end, or null if the path doesn't exist.
    private TrieNode traverse(String s) {
        TrieNode node = root;
        for (char c : s.toCharArray()) {
            int idx = c - 'a';
            if (node.children[idx] == null) return null;
            node = node.children[idx];
        }
        return node;
    }
}

class TrieNode {
    TrieNode[] children = new TrieNode[26];
    boolean isEnd = false;
}
```

### Why extract `traverse`?

`search` and `startsWith` both walk the same path — they only differ in what they check at the end. Extracting the walk into a private helper eliminates the duplication and makes the difference explicit at a glance.

### Why `children[c - 'a']` instead of a `HashMap`?

An array of size 26 is faster and simpler when the alphabet is fixed. Array access is O(1) with no hashing overhead and no collision handling. A `HashMap<Character, TrieNode>` would be appropriate if the character set were large or variable (e.g., Unicode), but for lowercase English letters, the array wins.

---

## Complexity

| | Insert | Search | startsWith |
|---|---|---|---|
| Time | O(m) | O(m) | O(m) |
| Space | O(m) per insert | O(1) | O(1) |

*m = length of the word or prefix*

Space for the entire trie is **O(n · m)** in the worst case (no shared prefixes), but much less in practice when words share common prefixes.

---

## Where Tries appear

- **Autocomplete** — jump to the prefix node, collect all words below it.
- **Spell checkers** — fast existence checks and nearest-word suggestions.
- **IP routing** — longest prefix match in networking tables.
- **Word Search II (LeetCode 212)** — prune DFS branches early using a Trie built from the word list.

---

## Summary

- A **Trie** stores strings as character-by-character paths from the root.
- Words sharing a prefix **share nodes** — that's where the efficiency comes from.
- Each node has a fixed-size `children[26]` array and an `isEnd` flag.
- All three operations (`insert`, `search`, `startsWith`) run in **O(m)** where `m` is the string length.
- The core pattern is a simple loop: for each character, compute the index, follow or create the child, advance the pointer.
