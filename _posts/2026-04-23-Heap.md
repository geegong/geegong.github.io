## What is a Heap?

A **Heap** is a complete binary tree that satisfies the **heap property**:

- **Min-Heap**: every parent is **smaller than or equal to** its children → the root is always the minimum.
- **Max-Heap**: every parent is **greater than or equal to** its children → the root is always the maximum.

```
Min-Heap             Max-Heap

       1                  9
      / \                / \
     3   2              7   8
    / \ / \            / \ / \
   7  4 5  6          2  4 5  6
```

The only guarantee is between a parent and its direct children — siblings have no required order. This is weaker than a sorted array, but that weakness is exactly what makes heaps so efficient.

## Storing a Heap in an Array

A complete binary tree maps perfectly onto an array with **no wasted space and no pointers**.

```
Index:  0   1   2   3   4   5   6
Value: [1,  3,  2,  7,  4,  5,  6]

       arr[0] = 1  (root)
       arr[1] = 3  arr[2] = 2  (level 1)
       arr[3] = 7  arr[4] = 4  arr[5] = 5  arr[6] = 6  (level 2)
```

Navigation formulas for a node at index `i`:

| Relation | Formula |
|---|---|
| Left child | `2*i + 1` |
| Right child | `2*i + 2` |
| Parent | `(i - 1) / 2` |

This is how Java's `PriorityQueue` and most heap implementations work under the hood.

## Core Operations

### Insert — `O(log n)`

Add the new element at the end of the array, then **bubble up** (also called *sift up*) until the heap property is restored.

```
Insert 1 into this min-heap:

Before:        2              Array: [2, 3, 5, 7, 4, 8]
              / \
             3   5
            / \ /
           7  4 8

Step 1 — append 1:            Array: [2, 3, 5, 7, 4, 8, 1]
              2
             / \
            3   5
           / \ / \
          7  4 8  1   ← new node at index 6

Step 2 — compare with parent arr[2]=5: 1 < 5, swap.
              2
             / \
            3   1
           / \ / \
          7  4 8  5

Step 3 — compare with parent arr[0]=2: 1 < 2, swap.
              1
             / \
            3   2
           / \ / \
          7  4 8  5

Heap property restored.
```

### Remove Min — `O(log n)`

Swap the root with the last element, remove the last element, then **bubble down** (also called *sift down*) the new root until the heap property is restored.

```
Remove min (root = 1) from:

       1              Array: [1, 3, 2, 7, 4, 5, 6]
      / \
     3   2
    / \ / \
   7  4 5  6

Step 1 — swap root with last, remove last:
       6              Array: [6, 3, 2, 7, 4, 5]
      / \
     3   2
    / \ /
   7  4 5

Step 2 — sift down: compare 6 with children 3 and 2. Min child is 2, swap.
       2
      / \
     3   6
    / \ /
   7  4 5

Step 3 — compare 6 with children 5. 5 < 6, swap.
       2
      / \
     3   5
    / \ /
   7  4 6

Heap property restored. Removed value: 1.
```

### Peek Min — `O(1)`

Just read `arr[0]`. The root is always the min (or max for a max-heap).

## Heapify — Building a Heap from an Array in `O(n)`

You might expect that inserting n elements one by one costs `O(n log n)`. But there's a smarter way: **heapify** builds a valid heap from an unsorted array in `O(n)`.

The idea: leaf nodes are already valid 1-element heaps. Start from the last non-leaf node (index `n/2 - 1`) and sift down each node toward the root.

```java
void heapify(int[] arr) {
    int n = arr.length;
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDown(arr, i, n);
    }
}
```

Why `O(n)`? Most nodes are near the bottom of the tree and barely move. The math works out to a linear bound even though each sift-down is `O(log n)`.

## Java's `PriorityQueue`

Despite the name, `PriorityQueue` is **not a traditional queue** — it is a **heap**.

A regular queue follows FIFO (First In, First Out): elements come out in the order they went in. A `PriorityQueue` ignores insertion order entirely and always returns the element with the highest priority (smallest value by default).

```
Regular Queue:    offer(3), offer(1), offer(2)  →  poll() = 3  (first in)
PriorityQueue:    offer(3), offer(1), offer(2)  →  poll() = 1  (smallest)
```

**The reason it achieves this efficiently is that it is backed by a heap internally.**

Java stores the heap as an array, but that array represents a binary tree using the same index formulas from earlier (`2i+1`, `2i+2`):

```
PriorityQueue after offer(3), offer(1), offer(2):

  Internal array: [1, 3, 2]

  This IS this binary tree:
         1         ← arr[0]
        / \
       3   2       ← arr[1], arr[2]
```

Every `offer()` triggers a **bubble up**, every `poll()` triggers a **sift down** — the exact same operations described above. The heap structure is just hidden inside the class.

### Three layers to keep straight

| Layer | What it is |
|---|---|
| **Heap** | A concrete data structure — a complete binary tree stored as an array |
| **Priority Queue** | An abstract concept (ADT) — "always give me the highest-priority element" |
| **Java `PriorityQueue`** | A concrete class that implements the Priority Queue ADT **using a heap** |

The name `PriorityQueue` describes the *behavior* (priority-based retrieval). The heap is the *implementation* that makes it efficient.

---

In Java, `PriorityQueue` is a **min-heap** by default.

```java
// Min-heap (default)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5);
minHeap.offer(1);
minHeap.offer(3);

minHeap.peek();  // 1 — view min without removing
minHeap.poll();  // 1 — remove and return min

// Max-heap — reverse the comparator
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
maxHeap.offer(5);
maxHeap.offer(1);
maxHeap.offer(3);
maxHeap.poll();  // 5
```

For custom objects, pass a lambda comparator:

```java
// Min-heap by second element of int[]
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[1] - b[1]);
```

| Method | Action | Time |
|---|---|---|
| `offer(e)` | Insert | O(log n) |
| `poll()` | Remove & return min | O(log n) |
| `peek()` | View min | O(1) |
| `size()` | Number of elements | O(1) |

## Heap vs Other Structures

| Operation | Sorted Array | Binary Search Tree | Heap |
|---|---|---|---|
| Find min/max | O(1) | O(log n) | O(1) |
| Insert | O(n) | O(log n) avg | O(log n) |
| Delete min/max | O(n) | O(log n) avg | O(log n) |
| Search arbitrary | O(log n) | O(log n) avg | O(n) |

The heap is unbeatable when you only need to **repeatedly get and remove the min or max** — it doesn't waste time maintaining full sort order.

## Classic Heap Patterns

### Pattern 1: Kth Largest Element

Maintain a **min-heap of size k**. The root is always the kth largest seen so far.

```java
// LeetCode 215 — Kth Largest Element in an Array
int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int num : nums) {
        minHeap.offer(num);
        if (minHeap.size() > k) {
            minHeap.poll();  // evict the smallest
        }
    }
    return minHeap.peek();  // root = kth largest
}
```

Why a min-heap for "largest"? Because you want to evict small numbers — and a min-heap puts the smallest at the top, making it cheap to remove.

### Pattern 2: Merge K Sorted Lists

Push the first element of each list into a min-heap. Repeatedly poll the minimum and push the next element from that list.

```java
// Each entry: [value, listIndex, elementIndex]
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[0] - b[0]);

// seed the heap
for (int i = 0; i < lists.length; i++) {
    if (lists[i] != null)
        pq.offer(new int[]{lists[i].val, i, 0});
}

while (!pq.isEmpty()) {
    int[] cur = pq.poll();
    // append cur[0] to result
    // push next element from the same list
}
```

### Pattern 3: Two Heaps (Median of a Stream)

Keep a **max-heap** for the lower half and a **min-heap** for the upper half. Balance their sizes so the median is always at one of the two roots.

```
Numbers seen so far: [1, 3, 5, 7, 9]

  lowerMax (max-heap)   upperMin (min-heap)
       3                      5
      / \                    / \
     1   (empty)            7   9

Median = (3 + 5) / 2 = 4.0
```

```java
PriorityQueue<Integer> lowerMax = new PriorityQueue<>(Collections.reverseOrder());
PriorityQueue<Integer> upperMin = new PriorityQueue<>();

void addNum(int num) {
    lowerMax.offer(num);
    upperMin.offer(lowerMax.poll());          // balance: push max of lower into upper
    if (upperMin.size() > lowerMax.size()) {  // keep lower >= upper in size
        lowerMax.offer(upperMin.poll());
    }
}

double findMedian() {
    if (lowerMax.size() > upperMin.size()) return lowerMax.peek();
    return (lowerMax.peek() + upperMin.peek()) / 2.0;
}
```

This two-heap trick is the key concept behind the **Median of Two Sorted Arrays** problem.

## Heap Sort — `O(n log n)`

A heap lets you sort in-place without extra space:

1. Build a max-heap from the array — `O(n)`.
2. Repeatedly swap the root (max) with the last element and sift down — `O(n log n)`.

```java
void heapSort(int[] arr) {
    int n = arr.length;

    // Build max-heap
    for (int i = n / 2 - 1; i >= 0; i--)
        siftDown(arr, i, n);

    // Extract elements one by one
    for (int i = n - 1; i > 0; i--) {
        swap(arr, 0, i);       // move current max to end
        siftDown(arr, 0, i);   // restore heap on the reduced array
    }
}
```

| Build heap (heapify) | O(n) | O(1) |
| Heap sort | O(n log n) | O(1) |

## Where Heaps Appear

- **Priority scheduling** — OS task schedulers pick the highest-priority task.
- **Dijkstra's algorithm** — always expand the unvisited node with the smallest known distance.
- **Top K problems** — any "find K largest/smallest" question.
- **Median tracking** — the two-heap pattern handles streaming medians.
- **Merge K sorted arrays/lists** — seed a heap with one element per list.
